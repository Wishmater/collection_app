Tags
	an item can have n tags
	tags can have child tags, creating a tree
	when adding a tag to an item, also add all its parents
		when removing a tag from an item, also remove all its children
		when reparenting a tag, need to update all items accordingly (out of scope for now)
	extra info in the tag:
		date created
		- item count
	??? should tag name be unique (? case sensitive)

Items
	usually an item corresponds to a file
		there might be folder/abstract items later, so it needs to be extensible
	extra info in the item:
	    int rating;
		int width
		int height
		DateTime dateAdded;
		DateTime dateLastSeen;
	File handling
		the collection has a root 
		the app has a file explorer view, starting from this root
			in this view it will be easy to see which files are not added as items (with color)
				folders will have a color for all added, one for none, one for some
			you can right-click to add an unadded file
				you can right-click a folder to add all its files
					optionally reccursively
					also, allow to set a series of tags to all the files in this folder
			--- how to make sure files won't be moved (which would be BAD)
